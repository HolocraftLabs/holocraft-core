# Holocraft Studio Structure

Holocraft is organized as a modular AI-native engineering ecosystem.

Each repository has a dedicated operational responsibility.

The ecosystem is designed for:

- scalability
- maintainability
- AI collaboration
- reusable infrastructure
- long-term growth

---

# Core Repositories

## holocraft-core

The central operating repository.

Contains:

- architecture rules
- engineering standards
- studio philosophy
- AI governance
- workflow systems
- operational documentation

This repository acts as the source of truth for the entire ecosystem.

---

## holocraft-ui

Shared design system and frontend foundation.

Contains:

- reusable UI components
- motion systems
- themes
- typography rules
- interaction patterns
- frontend utilities

This repository defines visual consistency across products.

---

## holocraft-web

Frontend applications and production websites.

Contains:

- marketing websites
- product frontends
- interactive web experiences
- client-facing systems

---

## holocraft-agents

Automation agents and AI workflows.

Contains:

- autonomous workflows
- AI orchestration systems
- automation pipelines
- task execution systems

---

## holocraft-assets

Shared creative resources.

Contains:

- branding assets
- design resources
- motion assets
- visual systems
- media libraries

---

# Operational Rules

All projects inside Holocraft should:

- follow shared engineering standards
- preserve architecture consistency
- reuse existing systems whenever possible
- avoid duplicated infrastructure
- prioritize maintainability

The ecosystem should evolve incrementally instead of through destructive rewrites.
