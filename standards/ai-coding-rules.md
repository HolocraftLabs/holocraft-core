# Holocraft AI Coding Rules

## Core Principle

AI assists engineering.

AI does not control architecture.

Human decisions override AI suggestions.

---

# AI Responsibilities

AI may:
- generate boilerplate
- assist refactoring
- improve readability
- accelerate iteration
- suggest optimizations

AI must NOT:
- rewrite entire systems unnecessarily
- change architecture without approval
- introduce random dependencies
- create duplicate components
- break consistency

---

# Stability Rules

Before modifying code AI must:
1. understand architecture
2. inspect folder structure
3. preserve naming conventions
4. avoid destructive refactors

---

# Dependency Rules

Avoid:
- unnecessary libraries
- trend-based packages
- unstable experimental tools

Prefer:
- native APIs
- stable ecosystem tools
- lightweight solutions

---

# Frontend Rules

AI should prioritize:
- readability
- responsiveness
- performance
- accessibility
- scalable composition

---

# Styling Rules

Avoid:
- random colors
- inconsistent spacing
- mixed design languages
- excessive effects

Use:
- spacing systems
- reusable tokens
- consistent hierarchy

---

# Component Rules

Prefer:
- small focused components
- reusable patterns
- isolated logic

Avoid:
- giant component files
- duplicated states
- prop drilling chaos

---

# Git Rules

AI must never:
- force push
- delete major systems
- overwrite stable branches

Without explicit approval.

---

# Communication Rules

AI responses should:
- be concise
- explain reasoning
- identify tradeoffs
- prioritize maintainability

---

# Long-Term Goal

The goal is not rapid code generation.

The goal is:
- sustainable systems
- scalable architecture
- maintainable engineering
- production-grade quality
